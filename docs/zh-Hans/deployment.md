# 部署

本文介绍 `ironclaw-zh` 的通用部署思路。目标是帮助你选择合适的运行方式，并在安全、自托管和可维护性之间取得平衡。

## 部署原则

- 优先本地优先
- 优先安全边界，而不是访问便利性
- 优先简单稳定的运行方式，再逐步增加能力
- 优先可回退、可备份、可升级

## 常见部署方式

### 本机原生运行

适合：

- 个人使用
- 调试与开发
- 快速验证

### Docker 部署

适合：

- 希望环境更一致
- 希望更容易迁移和复现
- 准备长期运行

### 服务器长期运行

适合：

- 长期在线运行
- 持续提供 Web Gateway
- 多设备自用访问

## 推荐部署顺序

1. 先跑通最小可用系统
2. 固定数据库与模型方案
3. 固定认证与访问方式
4. 建立日志与备份
5. 再逐步开启扩展能力

## 访问控制

- 默认只开放给可信用户
- 默认不直接公网暴露
- 默认保留最小访问面

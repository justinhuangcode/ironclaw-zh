# 安全说明

`ironclaw-zh` 不是一个重新设计安全模型的新项目。它的核心安全价值，主要来自上游 IronClaw 的架构设计，而不是中文化本身。

## 安全定位

本项目的安全目标是：

- 保持上游 IronClaw 的核心安全边界
- 不因中文本地化而削弱默认安全模型
- 让中文用户更容易理解部署边界、权限边界和风险点

## 安全边界来自哪里

核心安全能力主要包括：

- 沙箱化工具执行
- 基于能力声明的 WASM 工具模型
- 凭证宿主侧注入
- 提示注入防护与内容清洗
- 技能信任分级与工具降权
- 网关认证与最小暴露面设计

## 关键原则

### 提示词不是安全边界

模型是否听话，不是安全边界。

真正的安全边界来自：

- 认证
- 沙箱
- 工具能力限制
- 文件访问范围
- 凭证隔离

### 技能不是权限系统

技能负责行为提示，不负责真正的权限控制。

## 中文版的额外要求

- 不因为“中文化”而修改底层安全语义
- 不把“更易懂”误说成“更安全”
- 用户界面和文档可以中文化，内部协议和能力字段保持英文

# 配置

本文介绍 `ironclaw-zh` 的核心配置思路。目标不是列出所有底层参数，而是帮助你先把系统跑起来，再逐步开启更多能力。

## 配置原则

- 先最小配置，再逐步增加能力
- 优先使用安全默认值
- 用户文档解释配置含义，底层配置命名保持英文
- 区分“启动所必需的配置”和“增强能力配置”

## 配置分层

### 启动层配置

用于让程序先启动并完成初始化，例如：

- 数据库连接
- LLM 后端选择
- 基础认证信息

### 运行层配置

用于控制运行时行为和附加能力，例如：

- Gateway
- Sandbox
- Skills
- Routines
- Embeddings
- 扩展工具

## 推荐顺序

建议按以下顺序配置：

1. 数据库
2. LLM
3. 初始化流程
4. Gateway
5. Sandbox
6. Skills
7. 其他增强能力

## 技能与工具边界

`skills` 负责行为提示和任务风格。

真正的权限边界应由：

- 工具系统
- 能力声明
- 运行时限制
- 沙箱机制

来承担。

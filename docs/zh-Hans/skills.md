# 技能（Skills）

本文介绍 `ironclaw-zh` 中技能（skills）的定位、作用和最佳实践。

## 技能是什么

技能可以理解为：

- 面向特定任务场景的行为提示
- 帮助模型在某类任务中更稳定执行的上下文模块
- 提升中文默认体验和任务风格一致性的手段

技能不是：

- 任意代码执行插件
- 权限控制系统
- 可以替代工具系统和运行时边界的机制

## 在 `ironclaw-zh` 中为什么重要

中文发行版的价值，不只是把 UI 翻译成中文，而是让中文用户在常见任务里获得更自然、更稳定的默认体验。

这类体验通常来自：

- 中文任务表达习惯
- 中文写作与总结风格
- 中文用户常见工作流

## 设计原则

- 单一职责
- 窄而清晰
- 以行为规范为主
- 保持短小
- 中文友好，但不过度本地化

## 边界

- 技能负责行为提示
- 工具负责实际能力
- 运行时负责权限边界

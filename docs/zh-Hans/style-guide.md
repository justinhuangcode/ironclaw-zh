# 文档风格指南

本文定义 `ironclaw-zh` 的中文文档写作规范。

## 总体原则

- 中文为主，必要处补英文
- 先准确，再易懂
- 先清楚，再完整
- 用户可见层中文化，底层协议保持英文

## 中英文混排规则

第一次出现的关键概念采用：

- 中文（English）

例如：

- 本地优先（local-first）
- 自托管（self-hosting）
- 技能（skills）
- 沙箱（sandbox）

## 不翻译内容

- 环境变量名
- CLI 命令
- tool 名
- WIT 接口
- capabilities 字段
- 数据库字段
- 代码标识符

## 文风要求

- 直接
- 中性
- 工程化
- 可执行

避免：

- 过度营销
- 模糊承诺
- 一句话塞太多概念
